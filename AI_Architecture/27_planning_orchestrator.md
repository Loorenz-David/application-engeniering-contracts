# 27 — Planning Orchestrator Contract

## Definition

The planning orchestrator is the Tier 3 agent. It handles open-ended goals where the execution path cannot be determined at design time. It differs from a Tier 2 orchestrator in one fundamental way:

| | Tier 2 Orchestrator | Planning Orchestrator (Tier 3) |
|---|---|---|
| Decomposition | Written in code at design time | Generated by the LLM at runtime |
| Steps | Fixed and known | Discovered from the goal |
| Failure | Report and stop | Revise the plan and continue |
| Cost | Cheap — one LLM call per subagent | More expensive — plan + execution + optional revision |

Use Tier 3 when the user's goal is open-ended and the steps required to achieve it depend on what the system discovers along the way.

---

## The planning loop

The planning orchestrator runs three sequential phases:

```
1. PLAN     — one LLM call, no tools: produce a structured step-by-step plan
      ↓
2. EXECUTE  — run each step using the appropriate subagent; record results
      ↓ (if a step result changes what's needed)
   REVISE   — one LLM call: update the plan based on what was learned
      ↓
3. SYNTHESIZE — one LLM call: combine all results into a final user-facing response
```

Each phase is a distinct, explicit LLM call. The planning orchestrator does not use `AgentRunner`'s implicit tool loop — its control flow is explicit and inspectable.

## Plan data structures

```python
# ai/agents/planning_orchestrator/planner.py
from dataclasses import dataclass, field
from typing import Literal


@dataclass
class PlanStep:
    step_id: str
    # Sequential identifier. Format: "step_1", "step_2", etc.
    description: str
    # One sentence describing what this step achieves.
    subagent: str
    # Must be a key in PlanningOrchestratorConfig.available_subagents.
    task: str
    # The complete, self-contained task description passed to the subagent.
    # Must include all context the subagent needs — it has no memory of prior steps.
    depends_on: list[str]
    # step_ids that must complete before this step runs.
    # Keep sequential for now — parallel execution is not yet supported.
    status: Literal["pending", "running", "complete", "failed", "skipped"] = "pending"
    result: str | None = None
    error: str | None = None


@dataclass
class AgentPlan:
    goal: str
    steps: list[PlanStep]
    reasoning: str
    # Why these steps were chosen — visible in logs and streaming events.
    version: int = 1
    # Increments on each revision. Used in logs for tracing plan evolution.
```

## Phase 1 — Planning

The planner makes one LLM call with no backend tools. It uses a `create_plan` tool to force structured output.

```python
CREATE_PLAN_SCHEMA: dict = {
    "name": "create_plan",
    "description": "Output the execution plan for the user's goal.",
    "input_schema": {
        "type": "object",
        "properties": {
            "goal": {
                "type": "string",
                "description": "Restate the user's goal in one clear sentence.",
            },
            "reasoning": {
                "type": "string",
                "description": "Why these steps are necessary to achieve the goal.",
            },
            "steps": {
                "type": "array",
                "items": {
                    "type": "object",
                    "properties": {
                        "step_id": {"type": "string"},
                        "description": {"type": "string"},
                        "subagent": {"type": "string"},
                        "task": {"type": "string"},
                        "depends_on": {
                            "type": "array",
                            "items": {"type": "string"},
                        },
                    },
                    "required": ["step_id", "description", "subagent", "task", "depends_on"],
                },
            },
        },
        "required": ["goal", "reasoning", "steps"],
    },
}
```

```python
def _create_plan(self, user_message: str) -> AgentPlan:
    subagent_descriptions = "\n".join(
        f"- **{name}**: {config.system_prompt.splitlines()[1]}"
        # Pull the second line of the subagent's system prompt as its one-line description
        for name, config in self.config.available_subagents.items()
    )
    planning_context = (
        f"## Available subagents\n{subagent_descriptions}\n\n"
        f"## User goal\n{user_message}"
    )

    response = self.provider.chat(
        messages=[Message(role="user", content=planning_context)],
        tools=[CREATE_PLAN_SCHEMA],
        config=LLMConfig(
            **{**vars(self._llm_config()), "tool_choice": "required"},
        ),
    )

    args = response.tool_calls[0].arguments
    steps = [
        PlanStep(
            step_id=s["step_id"],
            description=s["description"],
            subagent=s["subagent"],
            task=s["task"],
            depends_on=s["depends_on"],
        )
        for s in args["steps"]
    ]

    if len(steps) > self.config.max_plan_steps:
        raise PlanningError(
            f"Plan has {len(steps)} steps — exceeds max_plan_steps ({self.config.max_plan_steps}). "
            "The goal may be too broad. Break it into smaller requests."
        )

    unknown_subagents = {s.subagent for s in steps} - set(self.config.available_subagents)
    if unknown_subagents:
        raise PlanningError(f"Plan references unknown subagents: {unknown_subagents}")

    return AgentPlan(
        goal=args["goal"],
        steps=steps,
        reasoning=args["reasoning"],
    )
```

## Phase 2 — Execution and revision

Steps execute sequentially in dependency order. After each step completes, the planner checks whether the result changes what subsequent steps need to do.

```python
def _execute_plan(
    self,
    plan: AgentPlan,
    agent_ctx: AgentContext,
) -> AgentPlan:
    revisions = 0

    while True:
        pending = [s for s in plan.steps if s.status == "pending"]
        if not pending:
            break

        # Find next runnable step — all dependencies complete
        runnable = [
            s for s in pending
            if all(
                any(dep.step_id == dep_id and dep.status == "complete" for dep in plan.steps)
                for dep_id in s.depends_on
            )
        ]
        if not runnable:
            # Dependency deadlock — a required step failed
            break

        step = runnable[0]
        step.status = "running"

        subagent_config = self.config.available_subagents[step.subagent]
        runner = AgentRunner(self.provider, subagent_config)
        result = runner.run(step.task, agent_ctx)

        if result.status == "complete":
            step.status = "complete"
            step.result = result.content

            # Check if this result changes what remaining steps need
            revised = self._maybe_revise(plan, step, agent_ctx)
            if revised and revisions < self.config.max_revisions:
                plan = revised
                plan.version += 1
                revisions += 1

        elif result.status == "clarification_needed":
            # Pause execution — propagate clarification upward
            # The current plan is embedded in the conversation history for resume
            raise _PlanClarificationSignal(
                clarification=result.clarification,
                plan=plan,
            )

        else:
            step.status = "failed"
            step.error = result.content or result.error
            # Attempt revision — a failed step may have an alternative path
            revised = self._maybe_revise(plan, step, agent_ctx)
            if revised and revisions < self.config.max_revisions:
                plan = revised
                plan.version += 1
                revisions += 1
            else:
                # Cannot recover — stop and report partial progress
                break

    return plan
```

### Revision check

```python
REVISE_PLAN_SCHEMA: dict = {
    "name": "revise_plan",
    "description": "Decide whether the plan needs updating given this step's result.",
    "input_schema": {
        "type": "object",
        "properties": {
            "needs_revision": {"type": "boolean"},
            "reasoning": {"type": "string"},
            "revised_steps": {
                "type": "array",
                # Same structure as create_plan steps.
                # Only include steps that are still pending — completed steps are immutable.
                "items": {"$ref": "#/$defs/step"},
            },
        },
        "required": ["needs_revision", "reasoning"],
    },
}


def _maybe_revise(
    self,
    plan: AgentPlan,
    completed_step: PlanStep,
    agent_ctx: AgentContext,
) -> AgentPlan | None:
    pending_steps = [s for s in plan.steps if s.status == "pending"]
    if not pending_steps:
        return None  # nothing left to revise

    revision_context = (
        f"## Original goal\n{plan.goal}\n\n"
        f"## Completed step\n"
        f"Step: {completed_step.description}\n"
        f"Result: {completed_step.result or completed_step.error}\n\n"
        f"## Remaining pending steps\n"
        + "\n".join(f"- {s.step_id}: {s.description}" for s in pending_steps)
    )

    response = self.provider.chat(
        messages=[Message(role="user", content=revision_context)],
        tools=[REVISE_PLAN_SCHEMA],
        config=LLMConfig(
            **{**vars(self._llm_config()), "tool_choice": "required"},
        ),
    )

    args = response.tool_calls[0].arguments
    if not args.get("needs_revision"):
        return None

    # Rebuild plan: keep completed steps, replace pending with revised steps
    completed = [s for s in plan.steps if s.status in ("complete", "failed", "skipped")]
    revised_pending = [
        PlanStep(**{**s, "status": "pending"})
        for s in args.get("revised_steps", [])
    ]
    return AgentPlan(
        goal=plan.goal,
        steps=completed + revised_pending,
        reasoning=args["reasoning"],
        version=plan.version,
    )
```

## Phase 3 — Synthesis

After all steps complete (or execution stops due to failure), the planner synthesizes all results into a single user-facing response.

```python
def _synthesize(
    self,
    goal: str,
    plan: AgentPlan,
    agent_ctx: AgentContext,
) -> str:
    completed = [s for s in plan.steps if s.status == "complete"]
    failed = [s for s in plan.steps if s.status == "failed"]

    synthesis_context = (
        f"## User goal\n{goal}\n\n"
        f"## Completed steps\n"
        + "\n".join(f"- {s.description}: {s.result}" for s in completed)
        + (
            f"\n\n## Failed steps\n"
            + "\n".join(f"- {s.description}: {s.error}" for s in failed)
            if failed else ""
        )
    )

    response = self.provider.chat(
        messages=[Message(role="user", content=synthesis_context)],
        tools=[],
        config=self._llm_config(),
    )
    return response.content or ""
```

## `PlanningOrchestrator` — full class

```python
# ai/agents/planning_orchestrator/planner.py

class PlanningOrchestrator:

    def __init__(self, provider: LLMProvider, config: PlanningOrchestratorConfig):
        self.provider = provider
        self.config = config

    def run(self, user_message: str, agent_ctx: AgentContext) -> AgentResult:
        try:
            plan = self._create_plan(user_message)
        except PlanningError as e:
            return AgentResult(status="failed", error=str(e), session_id=agent_ctx.session_id)

        try:
            plan = self._execute_plan(plan, agent_ctx)
        except _PlanClarificationSignal as signal:
            return AgentResult(
                status="clarification_needed",
                clarification=signal.clarification,
                session_id=agent_ctx.session_id,
            )

        content = self._synthesize(user_message, plan, agent_ctx)
        all_complete = all(s.status == "complete" for s in plan.steps)

        return AgentResult(
            status="complete" if all_complete else "failed",
            content=content,
            session_id=agent_ctx.session_id,
        )

    def stream(self, user_message: str, agent_ctx: AgentContext) -> Iterator[StreamEvent]:
        try:
            plan = self._create_plan(user_message)
        except PlanningError as e:
            yield StreamEvent(type="session_failed", data={"error": str(e)})
            return

        yield StreamEvent(type="plan_created", data={
            "goal": plan.goal,
            "reasoning": plan.reasoning,
            "steps": [
                {"step_id": s.step_id, "description": s.description, "subagent": s.subagent}
                for s in plan.steps
            ],
        })

        # Stream execution events per step
        for step in plan.steps:
            yield StreamEvent(type="plan_step_start", data={
                "step_id": step.step_id,
                "description": step.description,
                "subagent": step.subagent,
            })
            # ... execute step ...
            yield StreamEvent(type="plan_step_complete", data={
                "step_id": step.step_id,
                "status": step.status,
                "result": step.result,
            })

        content = self._synthesize(user_message, plan, agent_ctx)
        yield StreamEvent(type="session_complete", data={"content": content})

    def _llm_config(self) -> LLMConfig:
        base = self.config.llm_config or _default_llm_config()
        return LLMConfig(**{**vars(base), "system_prompt": self.config.system_prompt})
```

---

## Streaming events

These events extend the standard `StreamEvent` types from [20_streaming.md](20_streaming.md):

| Event type | When emitted | Key data |
|---|---|---|
| `plan_created` | After the planning phase | `goal`, `reasoning`, `steps[]` |
| `plan_step_start` | Before each subagent call | `step_id`, `description`, `subagent` |
| `plan_step_complete` | After each subagent call | `step_id`, `status`, `result` |
| `plan_revised` | After a revision | `version`, `reasoning`, `revised_steps[]` |

These events allow the client to show the user live progress — which step is running, which completed, when the plan changed.

## System prompt contract (`system_prompt.md`)

The planning orchestrator system prompt must contain four sections:

```markdown
# Planning Orchestrator

You coordinate complex, open-ended tasks by generating and executing a step-by-step plan.

## Planning rules
- State the goal clearly before listing steps.
- Each step must be self-contained — the subagent handling it has no access to prior steps.
- Include all context the subagent needs directly in the task description.
- Only use subagents from the available list. Never invent subagent names.
- Keep the plan to the minimum steps needed — do not add steps "just in case".

## Revision rules
- Revise only when a step result genuinely changes what subsequent steps need to do.
- Do not revise to correct a failed step into an identical step — that is a loop.
- Completed steps are immutable — only revise pending steps.

## Failure rules
- If a step fails and cannot be recovered, stop and report what was completed.
- Never silently skip a failed step. Always surface it in the final synthesis.

## Synthesis rules
- Summarize what was accomplished, what failed, and what the user should do next.
- Do not repeat step details verbatim — synthesize into a single coherent response.
```

---

## Clarification handling

When a subagent returns `clarification_needed`, the planning orchestrator applies the same resolution options as the Tier 2 orchestrator (see [10_orchestrator.md](10_orchestrator.md)):

1. **Answer from context** — check if the plan's already-completed step results contain the answer.
2. **Research subagent** — cross-domain questions spawn a minimal read-only subagent.
3. **Propagate upward** — intent questions reach the human.

The plan state is preserved in the `ClarificationRequest.conversation_history`. On resume, execution continues from the step that triggered the clarification.

## Router integration

The router (see [26_router_agent.md](26_router_agent.md)) dispatches to the planning orchestrator when no registered intent matches:

```python
# ai/router/router.py — in IntentRouter.route()

if decision.intent is None:
    planner = PlanningOrchestrator(self.provider, planning_agent.CONFIG)
    return planner.run(user_message, agent_ctx)
```

The planning orchestrator is not registered as an intent. It is the router's fallback, not a named destination.

## What the planning orchestrator must NOT do

- Call backend tools directly — all operations go through subagents.
- Generate plans with more than `max_plan_steps` steps — reject and return `PlanningError`.
- Reference subagents not declared in `available_subagents` — validate before execution starts.
- Revise completed steps — only pending steps may change.
- Revise more than `max_revisions` times — stop and report partial progress.
- Retry a failed step unchanged — a retry without change is a loop.
- Use `AgentRunner`'s implicit tool loop — the planning loop is explicit: plan → execute → revise → synthesize.
- Skip the synthesis phase even when all steps fail — always produce a human-readable summary of what happened.
