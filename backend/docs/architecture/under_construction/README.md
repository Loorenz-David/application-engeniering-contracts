# Under Construction Plans

Plans here are split into two types:

## `implementation/`

HOW plans — specific tasks, steps, contracts, and acceptance criteria for
building a feature or change. Each implementation plan should link back to
an intention plan that gives the WHY.

Naming: `PLAN_<slug>_<YYYYMMDD>.md`

## `intention/`

WHAT and WHY plans — goal-oriented documents that define the outcome you want
to achieve. A single intention plan can link to many implementation plans and
stays alive across multiple implementation cycles until the goal is reached.

Naming: `INTENTION_<slug>_<YYYYMMDD>.md`

---

The root `TEMPLATE_PLAN.md` is kept for backward compatibility. Prefer the
subfolder templates for new plans.
